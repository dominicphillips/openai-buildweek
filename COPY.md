# SOMETHINGS-ON copy system

This is the source of truth for product voice, interaction copy, and the design
guide’s conversational posture. Read it before writing or changing anything a
designer sees.

The first draft was generated with the repository copywriter workflow using
Grok, then edited against the live product. Model output is input, not authority.

## Position

SOMETHINGS-ON is a guided studio for emerging fashion designers.

Bring what catches your eye. Name what stays. Change one thing. Keep going.

It is not a clothing generator and it is not a blank canvas. It turns taste into
an authored decision that the designer can see, revise, and defend.

**The designer authors. The system labors.**

## Cultural grounding without imitation

The voice draws from three broad creative attitudes. These are design inputs,
not personas to copy.

| Reference | Translate into | Never reproduce |
| --- | --- | --- |
| Virgil Abloh | visible process, permission to begin, instructional labels, a restrained intentional edit | quotation-mark graphics, slogans, brand codes, or the claim that a small edit automatically makes a work original |
| Ye | conviction, reduction, useful scale, future-facing declarations | first-person impersonation, catchphrases, controversy, speech patterns, or grandiosity without a concrete action |
| Future | nocturnal mood, cadence, repetition as structure, self-authored identity, high/low tension | lyrics, ad-libs, slang pastiche, or a simulated artist endorsement |

The resulting voice must sound like SOMETHINGS-ON. Named cultural references do
not appear in product copy.

Grounding references:

- [Virgil Abloh at Harvard GSD](https://news.harvard.edu/gazette/story/2017/11/harvard-welcomes-virgil-abloh-for-gsd-lecture/)
- [Ye on creative risk and access](https://time.com/3823263/kanye-west-video/)
- [Future on dressing as self-expression and setting the day’s mood](https://www.gq.com/story/future-rapper-calvin-klein-milan-interview)

| --- | --- |
| Warmth | cool, never cold |
| Density | sparse |
| Certainty | high |
| Ceremony | light ritual |
| Expertise | plain first, technical when useful |
| Mood | night studio |

The room is dark. The instruction is clear. There is no performance pressure.

## Six rules

1. **Orient first.** Say where the designer is and what happens next.
2. **Give one action.** Choose, name, add, keep, change, compare, or open.
3. **Lower the stakes.** Incomplete is valid. Skip and Back remain visible.
4. **Mark progress.** Show stage, count, lineage, and the last change.
5. **Protect authorship.** The agent proposes and executes; the designer decides.
6. **Move one variable.** A smaller scope makes the decision legible.

Every line must do at least one job: orient, invite an action, reduce anxiety, or
mark progress. If it does none, remove it.

## Sentence system

- Prefer short declaratives.
- Put one idea in one sentence.
- Use periods more than commas.
- Use a question only when an answer changes the work.
- Use numbers for stages, counts, versions, and limits.
- Use labels such as `KEEP`, `CHANGE`, `REF / 01`, and `OBJECT / 001`.
- Use concrete fashion nouns: collar, hem, weight, rib, seam, drape, wash.
- Use verbs that move work: start, choose, set, add, keep, change, accept,
  reject, undo, compare, branch.
- No exclamation points.
- No dramatic ellipses.
- Contractions are fine when they sound natural.

## Words

### Use

notice · keep · change · evidence · reference · object · pile · version ·
branch · lineage · draft · compare · source · proportion · material ·
construction · finish · placement · author · studio · selected · incomplete ·
deliberate

### Use carefully

taste · attention · ritual · trait · hypothesis · analysis · iteration · mood ·
archive

### Do not use

unlock · supercharge · elevate · next-level · seamless · effortless · robust ·
innovative · magic · genius · iconic · viral · journey · manifest · sacred ·
hold space · trust the process · vibes · AI-powered

Also avoid:

- empty encouragement such as “Amazing work”;
- wellness theater that implies a breathing technique or treatment;
- “in the style of” requests or promises;
- fake homage or endorsement language;
- vague lifestyle instructions with no observable next action;
- paragraphs that contain more than one decision.

## Canonical onboarding copy

### Shared chrome

- Wordmark: `SOMETHINGS—ON`
- Progress: `{STAGE} · {NN} / 08`
- Stages: `Arrival · Sound · Pause · Attention · Taste · Object · Evidence · Studio`
- Back: `← Back`
- Footer: `Creative direction stays with you.`

### 01 / Arrival

- Eyebrow: `A guided studio for fashion ideas`
- Headline: `Start with what you notice.`
- Support: `Not a blank canvas. A place to find the first change.`
- Primary: `Begin ↗`
- Secondary: `Skip the ritual`

### 02 / Sound

The prior line, “Put on something you love,” is not approved. It does not tell
the designer whether the product will play music or what to do.

- Eyebrow: `Set the room`
- Headline: `Choose the sound.`
- Body: `Play one track that gets you working. Or keep it quiet.`
- Primary: `Sound is on ↗`
- Secondary: `Keep it quiet`
- Product behavior: no autoplay and no implied music integration.

### 03 / Pause

- Eyebrow: `A small pause`
- Headline: `Take 30 seconds.`
- Body: `Breathe at your own pace. We’ll keep the time.`
- Complete: `Ready when you are.`
- Timer controls: `Start · Pause · Resume · Skip`
- Note: `No technique. Just a pause.`

### 04 / Attention

- Eyebrow: `The idea can be incomplete`
- Headline: `You don’t need the whole idea.`
- Body: `Start with one detail that keeps your attention. We’ll follow it.`
- Primary: `Choose labels ↗`

### 05 / Taste

- Eyebrow: `Taste is a starting point`
- Headline: `Which labels do you return to?`
- Body: `Choose a few. They’re references, not rules.`
- Counter: `{NN} selected · aim for 3–5`
- Search: `Search labels or qualities`
- Empty result: `No labels match. Clear search or skip.`
- Primary: `Continue ↗`
- Secondary: `Skip for now`
- Missing requirement: `Pick at least one label, or skip.`

### 06 / Object

- Eyebrow: `Make it concrete`
- Headline: `What are we making first?`
- Body: `Choose one object. The collection can come later.`
- Suggested objects: `T-shirt · pants · shoe`
- Field: `OBJECT / 001`
- Placeholder: `e.g. a white T-shirt`
- Primary: `Set the object ↗`
- Secondary: `Start with a white tee`
- Missing requirement: `Name the object to continue.`

### 07 / Evidence

- Eyebrow: `Evidence before adjectives`
- Headline: `Show us what you mean.`
- Body: `Start from the local shelf, then add your own images or links.`
- Shelf: `LOCAL REFERENCES / 30`
- Shelf search: `Search object, material, construction, or mood`
- Empty result: `Nothing matches that search. Clear it or add your own.`
- Selection: `{NN} selected · up to 3`
- Drop zone: `Drop files or choose images`
- File note: `PNG, JPEG, WEBP · up to 3`
- Link label: `Or save a link card`
- Link note: `Links are saved, not fetched, in this prototype.`
- Primary with references: `Use these references ↗`
- Primary without references: `Open the studio ↗`
- Secondary: `Start without analysis`

### 08 / Threshold

- Eyebrow: `Your starting point`
- Headline: `Keep what matters. Change one thing.`
- Body: `Object centered. Conversation left. References within reach.`
- Primary: `Open the studio ↗`

## Studio copy

- Orientation: `Chat left. Object center. References around it.`
- Composer: `Keep ___. Change ___.`
- Empty chat: `State what stays. Then name one change.`
- Empty canvas history: `No edits yet. Name what stays. Name one change.`
- Viewport help: `Drag to arrange · scroll to scale`
- Version label: `VERSION / {NN}`
- Lineage: `FROM / {PARENT}`
- Change: `CHANGE / {SUMMARY}`
- Sources: `SOURCES / REF {IDS}`
- Actions: `Accept · Reject · Undo · Branch · Compare`
- Authorship marker: `AUTHORED CHANGES ONLY`

## Design guide behavior

The guide is studio labor. It proposes, executes, and records. It never claims
authorship.

Rules for every reply:

1. Ask for the invariant before the change.
2. Keep to one variable unless the designer explicitly expands the scope.
3. Restate `KEEP` and `CHANGE` before generating.
4. Translate brand names into neutral traits.
5. Do not offer or accept brand replication.
6. After a draft, offer a reversible choice.

Useful opener:

> What stays exactly as it is? What is the one thing you want to move?

Restatement:

> KEEP / boxy cut, heavy cotton
>
> CHANGE / rib collar to a clean band
>
> Generate the next draft?

Broad request:

> That is more than one move. Start with proportion, material, construction,
> finish, placement, color, or mood.

Clone request:

> I can use traits, not brand replication. Which quality should carry forward:
> proportion, material, finish, or placement?

After generation:

> DRAFT / 02 READY
>
> Changed: collar construction.
>
> Preserved: cut, weight, sleeve volume.
>
> Accept, reject, undo, or branch.

## System states

| State | Copy |
| --- | --- |
| Queue | `In queue.` |
| Generation | `DRAFT / {NN} · changing {VARIABLE} only` |
| Success | `DRAFT / {NN} READY` |
| Save | `Saving arrangement.` |
| Generic error | `That didn’t complete. Retry, edit the request, or undo.` |
| Image service | `Image service unavailable. Your setup is saved.` |
| Offline | `You’re offline. Edits stay local until connection returns.` |
| File type | `Use PNG, JPEG, or WEBP.` |
| File limit | `3 references max. Remove one to add another.` |
| Missing object | `Name the object to continue.` |
| Missing change | `Name one thing to change.` |

Errors are neutral and recoverable. Never blame the designer. Never pretend a
failed generation exists.

## Accessibility

- Every stage headline states the job in text.
- Back, Skip, silence, and reduced-motion paths remain exposed.
- No autoplay audio.
- Progress has the accessible name `Onboarding progress`.
- Brand and reference selections expose pressed state, not color alone.
- Icon-only actions have explicit labels.
- Generation state is text, not only a spinner.
- Motion never carries information by itself.
- Focus order follows the visible reading order.
- Use `role="alert"` only for errors that block progress.

## Review checklist

- [ ] Does every line orient, invite an action, reduce anxiety, or mark progress?
- [ ] Is the designer clearly the author?
- [ ] Is there only one requested change?
- [ ] Are counts, formats, and limits stated at the decision point?
- [ ] Can the designer skip, go back, reject, and undo?
- [ ] Does every error offer a next step?
- [ ] Are sources and version lineage visible?
- [ ] Is any phrase hype, wellness theater, vague lifestyle, or brand cloning?
- [ ] Does the copy remain useful with animation disabled?
- [ ] Does the guide restate `KEEP / CHANGE` before generation?
